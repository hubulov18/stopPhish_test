# Запуск

Проект одной командой в докер контейнере

`npm run compose:up`

Тесты:

1. Нужно поднять PostgreSQL (локально или через Docker).
2. `npm run test:integration -w backend`
3. Для unit-тестов: `npm run test:unit -w backend`

# Архитектура

Проект разделен на 2 workspace:
- `backend/`
- `frontend/`

Backend:

В Moleculer архитектура разделена на следующие сервисы:

- `api` - API-gateway
- `auth` - aутентиификация и авторизация с JWT
- `users` - логика работы с пользователями
- `categories` - логика работы с категориями заметок
- `notes` - логика работы с заметками и полнотекстный поиск по заметкам
- `system` - сервис для health-check проверки состояния приложения
Структура папок:
- `config` - конфиг БД для TypeORM
- `entities` - слой данных, сущности для TypeORM
- `migrations` - миграции для БД
- `scripts` - сиды для добавления тестовых данных в БД
- `services` - реализация бизнес логики вышеописанных сервисов
- `tests` - интеграционные и юнит тесты
- `types` - типы 
- `utils` - вспомогательные функции

ORM: TypeORM
База данных: PostgreSQL
При старте прилодения применяются миграции а также выполняется seed для добавления тестовых данных (пользователь категория и заметка)

Frontend:

Vue3 + Composition API + TypeScript, Vite + Tailwind

Основной CRUD функционал реализован а также почти все доп пункты.

Архитектура фронтенда включает следующие компоненты:

- `api` - здесь реализованнны функции для взаимодействия с API, регистрация авторизация, CRUD заметок и категорий. Также реализован полнотекстный поиск. 
Отдельно сделана обертка для запросов к API которая автоматически добавляет JWT токен в заголовки и может универсально использоваться для любых запросов к API.
- `components` - здесь должны находится переиспользуемые компоненты, но в масштабе данного проекта они не требуются,
добавил для целостной картины архитектуры.
- `pages` - здесь находятся страницы приложения которые отображаются по маршорутам указанным в router.ts. Всего 2 страницы автризация и страница заметок
- `stores` - глобальный state manager приложения, используется для хранения данных о текущем юзере и его токене а также о состоянии авторизации
- `types` - отдельно вынесенны типы данных, опять же в данном проекте не очень важно, скорее для демонстрации архитектурного стиля.
Также есть router который отвечает за навигацию между страницами и main.ts в котором происходит инициализация приложения.
Стили вынесенны на верхний уровень для удобства но в более сложном проекте их можно было бы перенести в папку с соответствующим экраном или компонентом.
